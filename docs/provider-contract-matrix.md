# Provider Read Contract Matrix

All provider operations are bounded, origin-pinned, read-only HTTP requests. Credentials are supplied only to the configured origin, redirects are rejected, response content is never included in an error, and no provider SDK or third-party runtime library is used.

| Provider | Requests and enterprise variants | Collection and response bounds | Normalization and retained evidence | Completeness and optional evidence | Error behavior |
| --- | --- | --- | --- | --- | --- |
| Jira | GET work search with token pagination; active plus recent completed default JQL; ticket fields, Atlassian rich text, newest comments, custom fields, and arbitrary attachment bytes | Page, issue, comment, field, attachment-count, per-file, cumulative-byte, response-byte, text, and timeout limits | Canonical Jira key/status category/namespace; recursive empty-value pruning retains meaningful `false` and `0`; comments render chronologically; attachments retain private path, size, MIME label, and SHA-256 without parsing | Failed later pages produce a partial result and retain prior catalog rows; individual comments, fields, and attachments report fetched/skipped/truncated/unavailable state | Authentication, permission, rate-limit, timeout, DNS, TLS, redirect, pagination, malformed payload, and response-bound failures use the shared redacted vocabulary; the previous catalog remains usable |
| GitLab | Numeric project ID or encoded group/subgroup path; MR detail, paginated notes/discussions/diffs/pipelines/jobs/tests, approvals, current-branch discovery, then Jira-key discovery | Per-endpoint pages and items; actor/note/discussion/diff/pipeline/job/test caps; per-text and global normalized context budgets; timeout and response-byte limits | One reconciled project/MR identity; canonical actors, notes, discussion resolution, mergeability, approvals, bounded diffs, pipelines, jobs, and test totals/cases | Component completeness flags and source/included counts; incomplete review reads retain last complete facets and cannot create false recovery | Ambiguous discovery is refused; unavailable permission-limited and rate-limited components retain fetched evidence plus the safe HTTP cause; origin, auth, permission, rate-limit, malformed, redirect, timeout, and bound failures are redacted |
| Jenkins | Classic job or build URL; multibranch parent and encoded branch job; build, causes, artifacts, changes, optional JUnit, Pipeline stages, and bounded `config.xml`; optional Jenkins-only TLS override | Response bytes, timeout, causes, artifacts, changes, affected paths, failed tests, stages, and config-text limits | Canonical build identity/status/URL; normalized test failures and stages; only literal `sonar.projectKey` and optional literal branch settings are retained from XML; raw XML and console logs are never stored | Missing JUnit, Pipeline API, or job config is normal unavailable optional evidence; invalid or permission-limited optional endpoints are partial; build completeness stays separate | Invalid multibranch selection is actionable; authentication, permission, rate-limit, timeout, DNS, TLS, malformed, origin, and response-bound failures expose no response body or credential |
| SonarQube | Quality-gate status, selected measures, unresolved issues with pagination, explicit project/branch selection, and safe dashboard link | Issue pages, issues per page, total issues, cumulative issue bytes, metric keys, response bytes, text, and timeout limits | Canonical project/branch; normalized gate conditions, measures, issues, impacts, tags, ranges, and only `id`/`branch` dashboard query values | Gate, measures, and issues have independent completeness; one failed component produces a partial branch context while successful components remain available | Changed totals/page metadata, malformed rows, permission limits, auth, rate limit, timeout, DNS, TLS, redirect, and byte limits produce bounded warnings or redacted failure without erasing prior complete evidence |

## Fixture and live evidence status

- Synthetic transport and normalization tests cover Jira pagination and partial retention, GitLab ID/path discovery and review/pipeline normalization, Jenkins classic/multibranch and unavailable optional APIs, and SonarQube branch/gate/issue behavior.
- Sanitized file fixtures are test-only inputs and contain `.example`/`.invalid` identities, no credentials, no employer identifiers, and no private response bodies.
- Live compatibility remains an explicit human gate. The checklist requires server version, authentication type, effective permissions, configured shape, unavailable endpoints, and complete/partial outcome without recording credentials or private provider data.
