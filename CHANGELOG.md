# Changelog

All notable changes to the Kronos preview are documented here.

## [Unreleased]

### Added

- Added **Use Event in Prompt** to the right-click menu for GitLab merge-request, Jenkins, and SonarQube Attention rows. It freezes only the selected retained transition into a private immutable Markdown/JSON pair, opens the standard review composer, and inserts one exact `[ATTENTION-*]` reference without provider refetch or terminal submission. Focused and activation regressions bring the dependency-free gate to 305 tests with 87.39% line, 78.48% branch, and 90.31% function coverage.
- Added a read-only Project Git-state dashboard with current branch, upstream drift, bounded local/remote branch inventory, staged and working-tree paths, and bounded diff. Project selection plus an explicit expanded/right-click action open the dashboard; **Open Source Control to switch** hands checkout and dirty-tree conflict decisions to VS Code instead of adding Git mutation authority to Kronos.
- Added a bounded Team Prompt Library with configurable local manifest files/directories and credential-free raw HTTPS Git manifests, private latest-good remote caching, origin-exact trusted GitLab authentication, searchable metadata, allowlisted session/project/Jira variables, full operator editing, credential redaction, and immutable exactly-once `[PROMPT-*]` placement into an explicitly managed terminal without submission. The library is available from Work, Sessions, Projects, ticket workspaces, Setup, and Doctor without adding another view or runtime dependency.
- Added an optional local-project nickname to guided Project Integration Setup and the Project action menu. Nicknames become the presentation name across Projects, Work, Jira selectors, Sessions, Attention, and new project launches; clearing one falls back to the stable catalog name without rewriting paths, ticket/session links, provider mappings, or Git state.
- Added a typed Claude launch permission-mode setting for Manual/default, Accept Edits, Plan, Auto, Don't Ask, and experimental Bypass Permissions. Raw permission flags remain blocked in the command setting; bypass requires a modal confirmation on every launch, and cancel/settings choices create no terminal or Session.
- Added a focused cross-view product-contract suite covering the four-view button hierarchy, canonical Setup/project ownership, ticket-free project launch, project-only Attention grouping, provider icon/severity language, rich Jira filtering, ticket-workspace action boundaries, and healthy Doctor behavior.
- Made Attention grouping strictly project-level: registered projects own their provider rows, and work without an explicit project appears under **Unassigned project** instead of a Jira-ticket/session group.
- Added an inline and expanded Project action that starts a ticket-free Claude session in the exact registered project directory and shows its observed branch in the new terminal title.
- Gave GitLab MR, Jenkins, and SonarQube Attention rows distinct theme-aware icons plus one shared state palette: green for healthy/recovered, yellow for warning/partial, and red for failed/blocked. Written provider and severity labels remain visible for accessible status recognition.
- Added a private bounded Context Basket for selecting Jira, GitLab MR, Jenkins/SonarQube, and local Git artifacts, reviewing provenance/freshness/completeness/size/hash/conflicts, explicitly refreshing sources, and placing one reference-only non-submitting bundle into an exact managed terminal.
- Added ephemeral bounded local evidence search across session titles, explicit Jira contexts, registered projects/branches, provider bindings, artifact labels, and audit-event summaries without reading or indexing terminal content.
- Added private local Markdown/JSON handoff bundles for up to 100 selected redacted context/audit references and hashes; exports never include terminal content, copy source payloads, or post to a provider.
- Added up to 20 explicit per-project Jenkins/SonarQube branch-routing profiles with exact MR-branch selection and an optional explicit fallback; profiles never infer ticket links or switch Git branches.
- Added a source-checked transition matrix for every declared GitLab MR, pipeline/job/test, Jenkins, and SonarQube transition, and extracted restart-safe current Attention projection from the view.
- Prioritized extension improvement roadmap covering identity, terminal lifecycle, provider reconciliation, Attention, cross-platform persistence, UX, maintainability, and release evidence.
- Dedicated Projects view with current branch, clean/dirty status, read-only Git loading, and existing project evidence actions.
- Recruiter-facing product overview, architecture, engineering metrics, and synthetic product renders.
- Branded Marketplace icon and preview metadata.
- Public-surface validation for tracked local state, machine paths, employer identifiers, private keys, and high-confidence token shapes.
- Security, support, and contribution guidance.

A direct discovery-to-registration regression brings the dependency-free gate to 304 tests and measured coverage to 87.43% lines, 78.59% branches, and 90.34% functions.

- Strengthened the dependency-free test gate from 248 to 256 behavior tests across 27 derived runners. - Healthy completed Jenkins builds now establish a first-observation Attention item just like healthy merge requests and SonarQube gates, including when Jenkins or SonarQube first becomes available after the other provider already established the combined CI baseline. Provider-read health now occupies the related GitLab MR, Jenkins build, or SonarQube project/branch stream, so partial/failure state replaces stale provider truth and recovery reveals the current result instead of leaving duplicate green rows.
